# Core Beliefs

FPVS Studio should feel like an experiment operations tool, not a general-purpose editor.

## Beliefs

- Returning users should reach launch readiness and the participant prompt with minimal
  navigation.
- First-time setup should be guided, linear, and focused on one decision area per step.
- Scientific/runtime status should be explicit. The app should not imply a project is
  launch-ready when validation says otherwise.
- Advanced controls are useful, but they should not compete with the normal setup path.
- Reusable UI styling and roles should go through `../../src/fpvs_studio/gui/components.py`
  through imports from GUI modules.
